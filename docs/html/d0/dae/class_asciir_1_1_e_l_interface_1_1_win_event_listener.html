◆ sendTermEvents()

void Asciir::ELInterface::WinEventListener::sendTermEvents ( )
protected

sends any terminal window related events to the event handler.

in contrast to the other send event methods, this one does not take any parameters, as it auto detects any changes related to the terminal status that has happened since the last call to this method.

it is done as such, as there is no way of detecting some events, like terminal window movement, without polling them.
