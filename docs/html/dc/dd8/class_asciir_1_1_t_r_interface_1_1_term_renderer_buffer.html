Detailed Description

@detail the windows implementation of the TermRendererBuffer uses two seperate console buffers, a writable, and a display buffer. the reason for this is that it is much faster to write to a console buffer that is not currently being displayed, thus a buffer swap system is implemented with the console buffers, where each console print, will result in the buffers being swapped.
